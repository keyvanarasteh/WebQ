# Paraglide-js: Strict Language Tag Types

**Context**: When configuring Paraglide-js for multi-language support (i18n) in SvelteKit layout files, dynamically pulling a language from `localStorage` often results in a generic `string` type.

**Problem:** Paraglide's generated `setLanguageTag` function is strictly typed via TypeScript to only accept specific available language literals (e.g., `"en" | "tr"`). Passing a generic `string` returned by `localStorage.getItem` causes a TypeScript compilation error during `bun run check`.

**Correct Implementation:**
1. Use an explicit `as` cast to constrain the localized string to the expected union type generated by Paraglide.

**Code Examples:**

❌ **INCORRECT:**
```svelte
<script lang="ts">
  import { setLanguageTag } from "$lib/paraglide/runtime";

  if (typeof localStorage !== "undefined") {
      setLanguageTag(localStorage.getItem('webq-lang') || 'en'); // Error: Argument of type 'string' is not assignable to parameter of type '"en" | "tr"'
  }
</script>
```

✅ **CORRECT:**
```svelte
<script lang="ts">
  import { setLanguageTag } from "$lib/paraglide/runtime";

  if (typeof localStorage !== "undefined") {
      setLanguageTag((localStorage.getItem('webq-lang') || 'en') as "en" | "tr");
  }
</script>
```
