# Complete React2Shell Honeypot UI

## Feature Description
The React2Shell Honeypot backend engine (`react_honeypot.rs` in `web-analyzer`) is fully functional and integrated via Tauri commands (e.g., `scan_react2shell`, `get_top_attackers`, `start_honeypot`). However, the WebQ frontend is missing crucial UI components to fully visualize and analyze the telemetry generated by the engine.

## Missing Components
The following routes and components need to be implemented in `/src/routes/react`:

1.  **Attacker Profiling (`/attackers/+page.svelte`)**
    *   Table view of `AttackerProfile` data.
    *   Display calculated `risk_score`, total requests, and geographic origin.
2.  **Signature Matrix Documentation (`/signatures/+page.svelte`)**
    *   Native rendering of the comprehensive list of 40+ regex-based attack signatures.
    *   Expandable accordions for each attack showing the Regex pattern from Rust, cURL examples, and remediation steps.
