# Feature Request: AI Crawler Readiness & WebMCP Dashboard

## Overview
The `web-analyzer` engine contains an advanced `geo_analysis.rs` module that goes beyond traditional OSINT. It calculates a "Geofencing AI readiness score" by searching for Model Context Protocol (WebMCP) endpoints (`/mcp.json`, `/.well-known/mcp`) and `llms.txt`. Furthermore, it parses `robots.txt` specifically against a hardcoded list of AI bots (GPTBot, ClaudeBot, OAI-SearchBot, PerplexityBot, etc.) to determine if the target is "Permissive" or "Restrictive".

This powerful, modern intelligence gathering capability is currently completely missing from the WebQ UI.

## Missing Engine Controls & Visualization
1. **AI Bot Matrix**: The UI needs a matrix showing the blocking status (Allowed, Blocked, Partially Blocked) for each specific LLM crawler.
2. **WebMCP Detection**: Visual alerts when `navigator.modelContext` or MCP REST endpoints are discovered, indicating the site is AI-agent compatible.
3. **LLMs.txt Viewer**: If `llms-full.txt` or `llms.txt` is found, provide a direct viewer to see the context the site feeds to AI agents.
4. **Overall Grade Card**: Display the calculated A-F grade and the 0-100 `geo_score` generated by the backend.

## Implementation Request
1. Create a new `AI Intelligence` or `Bot Landscape` tab in the Reconnaissance UI.
2. Add a `TechCategoryRadar` or matrix specifically for AI directives.
3. Ensure the Tauri backend actually calls `analyze_geo` and streams the `GeoAnalysisResult` back to the Svelte frontend.
