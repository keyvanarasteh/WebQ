# WebQ i18n Translation Guide

WebQ relies on the powerful [Paraglide SvelteKit](https://inlang.com/m/dxnzrydw/paraglide-sveltekit-i18n) framework to handle internationalization (i18n) directly within the frontend.

## The `messages/` Directory
All language files are stored in standard JSON key-value pairs underneath the `messages/` directory.

- `messages/en.json` (Base English)
- `messages/de.json` (German, etc.)

## How to Add Translations
1. Locate the UI string you wish to add in your respective language file (e.g., `messages/de.json`).
2. Add the corresponding key from `en.json` and translate its value.

Example:
```json
{
  "about_title": "Über WebQ",
  "clean_button": "Jetzt bereinigen"
}
```

3. If your language file does not exist yet, you can create it in the `messages/` folder and it will automatically be picked up by the compiler.
4. Paraglide automatically types and exposes these keys. You can use them directly in Svelte files via imports.

```svelte
<script>
  import * as m from '$lib/paraglide/messages';
</script>

<button>{m.clean_button()}</button>
```

Thank you for contributing your language to make WebQ accessible worldwide!
